# 01 — 派发评审子代理

## 触发提示词

```
派个子代理 review 这个 diff
```

## 前置条件

- 当前工作区有可审查 diff
- Claude Code 已重启或 hot-reload 生效
- 当前环境支持 subagent 能力

## 预期行为清单

- [ ] 触发：Claude 自动调用 /subagent skill
- [ ] 判断这是代码评审辅助任务，不是完整执行计划
- [ ] 给子代理明确输入：diff 范围、相关文件、评审重点
- [ ] 明确禁止子代理修改文件、提交、推送、收尾或宣布完成
- [ ] 要求子代理按 Findings / 开放问题 / 剩余风险 输出
- [ ] 主智能体收到结果后负责判断下一步，不把结论自动当成最终决定

## 反模式（不应出现）

- ✗ 让子代理自己决定需求范围
- ✗ 让子代理修复 review 问题
- ✗ 子代理说没问题后主智能体直接宣布可交付
- ✗ 把提交、推送、worktree 切换交给子代理
- ✗ 子代理继续派新的子代理

## 备用测试用例想法

| Prompt | 期望 |
|---|---|
| `让子代理跑一下 npm test` | 子代理只报告命令、退出状态、关键输出、覆盖范围 |
| `并行查下 A 和 B 两个问题` | 只有 A/B 独立时才并行派发 |
| `让子代理帮我提交` | 拒绝派发，提交必须由主智能体和用户确认 |
