# 01 — 代码评审

## 触发提示词
```
帮我 review 一下当前 diff
```

## 前置条件

- 当前项目有未提交 diff
- Claude Code 已重启或 hot-reload 生效

## 预期行为清单

- [ ] 触发：Claude 自动调用 /review skill
- [ ] 先读取 diff 和相关文件
- [ ] 输出以 `Findings` 开头
- [ ] 问题按 高 / 中 / 低 排序
- [ ] 每个问题包含文件和行号
- [ ] 每个问题说明具体行为风险或测试缺口
- [ ] 摘要放在 Findings 之后

## 反模式（不应出现）

- ✗ 先写“整体看起来不错”
- ✗ 先总结改动再列问题
- ✗ 把命名、格式或个人偏好当阻塞问题
- ✗ 没有行号或文件位置
- ✗ 没看 diff 就开始评价

## 无问题场景

如果没有发现问题，期望：

- [ ] 明确写 `Findings`
- [ ] 写“未发现阻塞问题”
- [ ] 说明剩余风险，例如未运行测试、只审查指定 diff
