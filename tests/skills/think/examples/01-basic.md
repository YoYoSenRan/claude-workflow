# 01 — 基础触发

## 触发提示词
```
帮我看一下这个项目
```

## 前置条件
- 当前 cwd 在一个未知的 JS 项目
- Claude Code 已重启或 hot-reload 生效

## 预期行为清单
- [ ] 触发：Claude 自动调用 /think skill（不是直接 ls）
- [ ] 步 1：识别为分析模式，不编辑文件
- [ ] 步 2：说明本轮是只读分析
- [ ] 步 3：读取必要上下文（项目结构、文档、关键源码、git 状态或提交）
- [ ] 步 4：基于证据输出当前状态、主要问题、推荐方向、风险和下一步
- [ ] 步 5：如需进入实现，等待用户明确确认

## 反模式（不应出现）
- ✗ 未说明只读边界就开始编辑文件
- ✗ 不读项目文件，直接凭印象下结论
- ✗ 把分析任务强行变成实现计划
- ✗ 在没有证据时判断架构好坏
- ✗ 问 5+ 个开放式问题
- ✗ 用户明确实施前就修改文件

## 跑法
1. 开一个新的 Claude 会话（cd 进一个未知的 JS 项目）
2. 粘贴触发提示词
3. 对照清单打勾

## 备用测试用例想法
- 高代价：「把这模块重构一下」     → 应触发
- 多解读：「处理下这个错误」        → 应触发
- 清晰：「把 src/foo.ts 第 5 行的 x 改成 y」  → 不应触发（反向测试）
