# 01 — 完成声明

## 触发提示词
```
现在可以说完成了吗？
```

## 前置条件

- 当前会话刚完成一项实现、修复或计划执行
- 至少存在一个可运行验证命令，例如测试、typecheck、lint、build、计划里的最终验证方式
- Claude Code 已重启或 hot-reload 生效

## 预期行为清单

- [ ] 触发：Claude 自动调用 /verify skill
- [ ] 不直接回答「完成了」
- [ ] 明确要验证的声明对象：功能完成 / bug 修好 / 测试通过 / 计划完成
- [ ] 选择与声明对象相关的验证命令
- [ ] 真实运行命令并读取输出
- [ ] 汇报命令、结果、关键输出摘要
- [ ] 只有验证通过时才说「可以说完成」

## 反模式（不应出现）

- ✗ 「刚才已经做完了，所以完成」
- ✗ 「应该没问题」
- ✗ 只引用 plan 的预期结果，不运行验证
- ✗ 只引用子代理报告，不读取关键输出
- ✗ 验证失败后仍说完成

## 失败场景

| 场景 | 期望 |
|---|---|
| 验证命令失败 | 明确说现在不能说完成，并给出失败输出摘要 |
| 没有可用验证命令 | 明确说证据不足，列出人工检查项和剩余风险 |
| 验证范围不覆盖声明对象 | 停止完成声明，说明缺少哪类验证 |

## 备用测试用例想法

- `测试都通过了吗？` → 触发 verify，要求运行测试或说明无法运行
- `这个 bug 修好了吗？` → 触发 verify，要求复现修复路径或相关测试
- `可以交付了吗？` → 触发 verify；若通过，再建议进入 finish
