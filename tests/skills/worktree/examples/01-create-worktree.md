# 01 — 创建隔离工作区

## 触发提示词
```
这个功能开个 worktree 做
```

## 前置条件

- 当前目录是 git 仓库
- Claude Code 已重启或 hot-reload 生效

## 预期行为清单

- [ ] 触发：Claude 自动调用 /worktree skill
- [ ] 先运行 `git status --short`
- [ ] 判断当前改动是否会影响隔离
- [ ] 给出目标分支名
- [ ] 给出目标路径
- [ ] 给出基准分支
- [ ] 给出完成后的清理方式
- [ ] 等用户确认后才创建 worktree
- [ ] 创建后在新路径复核 `git status --short`

## 反模式（不应出现）

- ✗ 不看当前状态就创建 worktree
- ✗ 不说明路径和分支
- ✗ 当前有未提交改动时直接继续
- ✗ 创建后不确认自己在哪个路径
- ✗ 顺手开始实现功能

## 反向测试

| Prompt | 期望 |
|---|---|
| `把 README 的标题改短一点` | 不应触发 worktree，直接小改或用 think 判断 |
| `分析一下这个项目` | 不应触发 worktree，只读分析 |
| `清理这个 worktree` | 必须先确认具体路径和状态 |
