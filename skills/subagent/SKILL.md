# 子代理调度

subagent 只负责判断和组织子代理任务。不执行计划、不提交代码、不切换工作区，也不替用户批准方向。

声明用 A 风格：主智能体进行打头 `> **🔧 subagent** — 派遣<目的>`；整合子代理结果后由主智能体收口。

<HARD-GATE>
不要把主智能体必须负责的决策交给子代理：用户确认、需求批准、计划执行权、提交 / 推送 / 丢弃、工作区切换、最终完成声明。
</HARD-GATE>

## 何时使用

用户明确要求派 subagent / agent review；需要并行调查多个互不依赖的问题；需要独立 review 计划、代码或验证输出；任务较大但某子问题可只读调查；主智能体需省上下文只让子代理收集事实。

不使用：清晰小改；单一问题主智能体当前就能处理；下一步依赖子代理结果且自己必须先知道答案才能继续；需要用户确认或高风险写操作；当前环境没有可用 subagent 能力。

## 可派发任务

适合：方案调研、计划草案（不批准）、计划评审、根因调查、验证执行、代码评审——都只读或只产出建议。

不适合：HARD-GATE 列出的全部主智能体决策（定范围、批准、改计划状态、执行完整计划、提交推送丢弃、worktree 操作、宣布完成）。

## 调度流程

1. **确认目标** — 子代理要回答什么具体问题。
2. **判断独立性** — 子任务能否独立完成，不阻塞主智能体下一步。
3. **选择模式** — 只读调查 / 计划草案 / 评审 / 验证 / 代码评审。
4. **限定输入** — 给必要文件、路径、命令、约束和输出格式，不泄露无关上下文。
5. **明确禁止项** — 写明不得修改文件、不得提交、不得宣布完成。
6. **继续主线** — 子代理运行时主智能体做不重叠的工作。
7. **整合结果** — 主智能体复核关键证据，决定下一步 skill。

下一步必须等子代理结果时，少派或不派，主智能体直接做关键路径任务。

## 提示模板

```text
你是被派遣的子代理。只完成下面这个任务，不要启动主流程，不要再派子代理。

任务：<一句话目标>
输入：
- 文件 / 路径：<精确路径>
- 需求 / 计划 / 错误：<必要上下文>
- 可运行命令：<如有>
边界：
- 不修改文件，除非任务明确要求。
- 不提交、不推送、不切换工作区。
- 不宣布最终完成或可交付。
- 只返回证据和结论。
输出格式：
结论：... / 证据：... / 问题：... / 建议下一步：...
```

代码评审子代理输出用 `发现项 / 开放问题 / 剩余风险`；验证子代理输出用 `命令 / 退出状态 / 关键输出 / 覆盖范围 / 未覆盖风险`。

## 与其他 skill 的关系

| 场景 | 下一步 |
|---|---|
| 子代理发现需求不清 | 回 `think` |
| 子代理发现计划问题 | 回 `plan` 修订 |
| 子代理发现未知 bug | 回 `debug` |
| 子代理返回验证输出 | 主智能体用 `verify` 判断是否足够 |
| 子代理完成评审 | 主智能体决定是否修复、计划或收尾 |

## 警示信号

| 念头 | 现实 |
|---|---|
| "让子代理自己判断范围" | 范围由主智能体和用户确认。 |
| "子代理说通过就算完成" | 最终完成声明属于主智能体的 `verify`。 |
| "派出去就不用管了" | 主智能体必须整合结果和承担决策。 |
| "子代理也可以继续派子代理" | 禁止递归调度。 |
| "并行越多越快" | 只有独立任务才并行；关键路径本地处理。 |

## 何时不激活

- 用户没有要求 subagent，且任务能直接处理。
- 当前任务是轻量小改。
- 当前正在执行已批准 plan，应该用 `execute`。
- 当前是收尾、提交、推送或 worktree 操作。
