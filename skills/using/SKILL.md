# 技能路由

收到消息时，先判断有没有技能适用。不确定就调——调了发现不适用再跳过。

## 上次会话

如果 `.claude/last-session.md` 存在，读一下。它记录了上次会话做了什么、改了什么、还有什么没做。用户问"继续"或开始相关任务时，用这个信息恢复上下文。

## 路由表

| 用户想要 | 调什么 |
|---|---|
| 新功能、新组件、改行为 | think |
| 修 bug、测试失败、异常 | debug |
| 初始化项目工作流 | setup |
| 创建或编辑技能 | skill |

## 多个都可能适用时

1. 先调流程类（think、debug）——决定怎么做
2. 再调实现类——指导执行

think 确认的是方向，plan 确认的是执行步骤。用户没明确说"直接做"时，plan 之后要等用户确认。

## 技能加载后

- 严格按内容执行，不裁剪、不自己改意思
- 技能说必须严格遵守的（TDD、debug）就一步不省
- 技能说可以灵活的就适配当前情况
- 技能有 checklist 就用 TodoWrite 落地

## 对用户说话时

不提技能名、不说"加载"、不暴露工作流。只说你在做什么。
