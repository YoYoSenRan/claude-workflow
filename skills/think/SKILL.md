# 对齐用户意图

用户 prompt 模糊或多解读时,先停手对齐,再做任何动作。30 秒对齐 = 省 1 小时返工。

<HARD-GATE>
检测到 prompt 模糊时,**禁止**调用任何工具(Read / Edit / Bash / Skill 等),**禁止**做任何"先看一下""先试一下"的探索性动作。

必须先完成: 复述理解 → 提 W 问题 → 等用户确认。确认后才能进入执行。

这条不可协商。"先快速看一下"不是例外,是违规。
</HARD-GATE>

## 反模式: "这个简单不用对齐"

最常掉的坑。每次想"这 prompt 简单,直接干就行"时,就是该对齐的时刻。

模糊 prompt 没有简单的——简单的都是清晰的(具体文件 + 具体动作)。"看下""改改""处理一下"全都不简单——它们藏了多个可能的执行路径,选错就白干。

## 清单

按顺序完成,不可跳:

1. **扫歧义** — 静默扫 prompt,看是否命中歧义信号
2. **暂停** — 命中即停,不调任何工具
3. **复述** — 1-2 句具体复述你的理解
4. **提问** — 选 2-3 个最相关的 W 问题,**给选项**
5. **等确认** — 用户回答清晰后才执行

## 歧义信号 (任一命中即触发)

| 信号 | 例子 |
|---|---|
| 指代不明 | "这块" / "那个" / "处理一下" / "改改" |
| 范围模糊 | "看下" / "做个 X" 没说边界 |
| 多解读 | 至少能想出 2 个合理理解 |
| 高代价决策 | 删除 / 重命名 / 改架构 / 改公共 API |
| 复合动词 | "重构 + 优化" / "改造 + 文档化" |

都不命中 → 跳过本 skill,直接执行。

## 流程图

```
prompt 来
   ↓
扫歧义信号 ─── 都不命中 ───→ 跳过 skill,执行
   ↓ 任一命中
暂停 (不调工具)
   ↓
复述理解
   ↓
问 2-3 个 W (给选项)
   ↓
用户回答清晰? ── 不清晰 ──→ 再问 1 轮 (最多 2 轮)
   ↓ 清晰
执行
```

## 复述要求

复述要**具体**,不能空泛。空泛 = 没真理解。

❌ "你想优化代码"  
✓ "你想把 src/api.ts 的同步 fetch 改成异步,但保持 callsite 不变"

## W 提问

挑跟当前歧义最相关的 **2-3 个**,**给选项**,不要纯开放式:

| W | 问什么 |
|---|---|
| What | 具体要做什么? (动作 / 产出) |
| Why | 目的是啥? (问题 / 业务驱动) |
| Where | 范围在哪? (文件 / 模块) |
| Who | 给谁用? (受众 / 调用方) |
| When | 优先级 / 截止? |
| How | 方式偏好? (技术 / 风格) |
| Not | 明确**不**要什么? |

❌ "你想要什么?"  
✓ "你想 A: 全部重写, 还是 B: 只改 X, 还是 C: 加个新文件?"

## 等确认时的处理

- 回答**清晰** → 执行
- 回答**仍模糊** → 再问 1 轮 (最多 2 轮)
- 用户说**"按你想的"** → 给最可能的具体方案,问"是这个吗",**不擅自决定**

## 危险信号

念头出现 → 立刻停下,是偷懒借口:

| 内心戏 | 真相 |
|---|---|
| "用户大概是想 X" | 大概 ≠ 确定。先确认。 |
| "我先 ls 看下再决定" | ls 是工具调用,违反 HARD-GATE。先停。 |
| "这事简单不用对齐" | 简单的事更容易做错方向。 |
| "上次他都这么做" | 上次 ≠ 这次。每次独立判断。 |
| "我先试一种,错了再改" | "试" = 工具调用 = 违规。 |
| "问太多用户会烦" | 问 2-3 问 30 秒,做错方向 1 小时。 |
| "他说'对齐意图'就是让我别问" | 不,是让你**用**这套对齐流程。 |

## 何时不激活

满足以下 → 跳过 think:

- prompt 自带具体文件 / 函数 / 行号
- 重复性命令 (`npm test` / `git status` / 格式化)
- 用户明确说"快速试一下 / 不用对齐"
- subagent 任务 (任务已由主 agent 拆好)
- 事实查询 ("这函数返回什么?")
- 用户在 PlanMode 后已审计划,进入执行阶段

## 核心原则

- **HARD-GATE 不可破** — 没有"先快速看一下"的例外
- **复述具体** — 模糊复述 = 没理解,白对齐
- **选项提问** — 开放式问题用户也不知道怎么答
- **最多 2 轮** — 死磕不行,让用户重新表达
- **不擅自决定** — "按你想的"也要先讲方案再问
