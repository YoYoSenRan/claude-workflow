# 执行计划

读取已批准的计划，先做 critical review，再按任务执行。计划是契约，不是建议。

<HARD-GATE>
遇到阻塞点必须停：计划含糊、文件不存在、命令无法运行、测试失败反复出现、实现需要超出计划范围。

禁止跳步骤、改计划、猜测意图、编造预期输出。
</HARD-GATE>

## 前置条件

进入本 skill 前必须满足：

- 用户明确要求执行计划；
- 存在书面计划文件，或用户贴出了完整计划；
- 计划任务有明确 涉及文件 / 步骤 / 运行 / 预期 / 验证 / 完成标准。

如果没有计划，停止并转回 `plan`。

---

## 清单

按顺序完成：

1. **读取计划** — 一次性读取完整计划。
2. **critical review** — 检查路径、命令、任务顺序、缺失步骤和含糊点。
3. **建立 TodoWrite** — 每个任务一项，不为每个步骤建任务。
4. **逐任务执行** — 只按计划列出的文件和步骤做。
5. **逐步验证** — 每个运行命令都要真实执行并读取输出。
6. **遇阻即停** — 报告具体任务 / 步骤 / 输出，不猜。
7. **完成后转 verify** — 需要完成声明前，进入 `verify` 或执行等价新鲜验证。

---

## critical review

执行前先检查：

- 计划文件是否存在；
- 每个任务是否有清晰目的；
- 涉及文件路径是否存在或标明新建；
- 步骤是否能落地；
- 运行命令是否存在；
- 预期是否具体；
- 验证和完成标准是否可验证；
- 任务顺序是否依赖倒挂；
- 是否需要修改计划未列出的文件。

任一项失败，停止并报告：

```text
计划阻塞：任务 N / 步骤 M 存在 <问题>。
原因：<为什么会导致执行失败或偏离需求>。
建议：回到 plan 修正后再执行。
```

---

## 执行规则

- 每次只执行当前任务。
- 当前任务完成并验证后，才进入下一个任务。
- 不主动添加计划外功能。
- 不主动重构计划外代码。
- 不修计划外 bug；只在 `> Note:` 中提示。
- 只允许更新计划里的 `状态` 和 checkbox；不得改写任务内容，除非用户明确要求回到 `plan` 修订。
- 如果验证失败，允许围绕当前步骤做一次最小修正；仍失败就停。

---

## 阻塞点

| 阻塞点 | 处理 |
|---|---|
| 计划路径不存在 | 停，要求提供正确计划或回到 `plan` |
| 文件路径不存在且未标新建 | 停，报告路径问题 |
| 步骤含糊 | 停，列出候选解读 |
| 运行命令不存在 | 停，报告命令问题 |
| 预期不具体 | 停，要求修计划 |
| 验证或完成标准不可验证 | 停，要求修计划 |
| 测试失败超过 1 次最小修正 | 停，报告输出 |
| 需要计划外文件 | 停，说明超出范围 |
| 发现 bug 根因不明 | 转 `debug` |

---

## subagent 边界

当前 `execute` 不承担完整 subagent-driven-development。

原因：

- 当前仓库还没有配套 `agents/`；
- 没有 implementer / spec reviewer / code reviewer 的完整 prompt；
- 没有自动化验证 subagent handoff 行为。

如果任务过大，不适合 inline 执行，停止并建议：

```text
这个计划不适合 inline execute。建议先补 subagent 工作流或把计划拆小。
```

不要假装已经做过 subagent review。

---

## 完成条件

一个任务完成必须满足：

- 计划列出的文件已按步骤处理；
- 计划列出的运行命令已执行；
- 输出符合预期；
- 没有引入计划外变更；
- TodoWrite 中该任务标为 completed。

全部任务完成后：

- 不直接声称“完成”；
- 先进入 `verify`，或运行计划指定的最终验证命令；
- 有新鲜证据后再汇报。

## 警示信号

| 念头 | 现实 |
|---|---|
| "这一步大概是这个意思" | 大概就是阻塞点，停。 |
| "计划漏了，我顺手补上" | 顺手就是改计划，停。 |
| "测试失败像环境问题" | 没证据不能跳过。 |
| "先做后面的任务" | 顺序可能有依赖，不能跳。 |
| "这个适合派子代理" | 当前没有完整 subagent 工作流，先停或拆小。 |

## 何时不激活

- 没有书面计划，应该用 `plan`。
- 需求还没确认，应该用 `think`。
- 是 bug 根因排查，应该用 `debug`。
- 只是单文件小改，不需要执行计划。
