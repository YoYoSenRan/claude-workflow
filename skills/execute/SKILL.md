# 执行方案

## 概述

载入方案，批判性地审阅，执行全部任务，完成后汇报。

**注意：** 告诉用户：Claude Workflow 在能调用子代理时效果好得多。若运行在支持子代理的平台（例如 Claude Code 或 Codex），其工作质量会显著更高。若可用子代理，请改用 claude-workflow:subagent，而不是这个技能。

## 流程

### 步骤 1：载入并审阅方案
1. 阅读方案文件
2. 批判性审阅 —— 找出对方案的疑问或顾虑
3. 若有顾虑：开始前与用户沟通
4. 若无顾虑：建立 TodoWrite 并开始执行

### 步骤 2：执行任务

对每一项任务：
1. 标记为 in_progress
2. 严格按每个步骤执行（方案的步骤已被切成小块）
3. 按要求运行验证
4. 标记为 completed

### 步骤 3：完成开发

所有任务完成并核验后：
- **下一步：** 使用 claude-workflow:finish
- 按那个技能去核验测试、给出选项、执行选择

## 何时停下并求助

**立即停止执行的情况：**
- 遇到阻塞（依赖缺失、测试失败、说明不清）
- 方案存在关键缺口，无法启动
- 你看不懂某条说明
- 验证反复失败

**与其乱猜，不如请求澄清。**

## 何时回到先前的步骤

**回到审阅（步骤 1）的情况：**
- 伙伴根据你的反馈更新了方案
- 基本方法需要重新思考

**不要强行突破阻塞** —— 停下并询问。

## 牢记
- 先批判性审阅方案
- 严格按方案的步骤执行
- 不要跳过验证
- 方案要求时引用相应技能
- 阻塞时停下，不要猜
- 未经用户明确同意，绝不要在 main/master 分支上开始实现

## 集成

**必需的工作流技能：**
- **claude-workflow:worktree** —— 确保隔离工作区（创建一个或核验已有的）
- **claude-workflow:plan** —— 编写本技能要执行的方案
- **claude-workflow:finish** —— 在所有任务后收尾开发
