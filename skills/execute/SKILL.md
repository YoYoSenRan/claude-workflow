# 执行方案

<HARD-GATE>
每个步骤执行完毕后，必须运行对应的验证命令并确认通过，才能进入下一步。不允许批量执行多步后再统一验证。

未经用户明确同意，绝不要在 main/master 分支上开始实现。
</HARD-GATE>

载入方案，批判性地审阅，执行全部任务，完成后汇报。

**注意：** 告诉用户：Claude Workflow 在能调用子代理时效果好得多。若运行在支持子代理的平台（例如 Claude Code 或 Codex），其工作质量会显著更高。若可用子代理，请改用 claude-workflow:subagent，而不是这个技能。

## 何时使用

- 有一份书面的实施方案需要执行时
- 在带评审检查点的单独会话中执行时
- 不支持子代理或用户选择内联执行时

若可用子代理，请改用 claude-workflow:subagent。

## 流程

### 步骤 1：载入并审阅方案

1. **定位方案文件：** 如果用户指定了路径，直接读取。如果没有指定，扫描 `docs/plans/` 目录，找包含 `- [ ]` 的文件。找到多个时列出让用户选择。
2. 阅读方案文件
3. **检测恢复场景：** 如果 plan 文件中已有 `- [x]` 步骤，说明是中断恢复。汇报已完成的步骤数，从第一个 `- [ ]` 的步骤继续。
3. 批判性审阅 —— 找出对方案的疑问或顾虑
4. 若有顾虑：开始前与用户沟通
5. 若无顾虑：建立 TodoWrite 并开始执行

### 步骤 2：执行任务

**plan 文件是进度的唯一真相来源。** TodoWrite 仅作会话内进度显示，丢失不影响恢复。

对每一个步骤：
1. 在 TodoWrite 中标记为 in_progress
2. 严格按步骤内容执行
3. 运行步骤中的验证命令
4. **验证通过：** 把 plan 文件中该步骤的 `- [ ]` 改为 `- [x]`，在 TodoWrite 中标记 completed
5. **验证失败：** 在 plan 文件该步骤下追加 `  - ❌ failed: <错误信息>`，停下求助

### 步骤 3：完成开发

所有任务完成并核验后：
- **下一步：** 使用 claude-workflow:finish
- 按那个技能去核验测试、给出选项、执行选择

### 何时停下并求助

**立即停止执行的情况：**
- 遇到阻塞（依赖缺失、测试失败、说明不清）
- 方案存在关键缺口，无法启动
- 你看不懂某条说明
- 验证反复失败

**与其乱猜，不如请求澄清。**

### 何时回到先前的步骤

**回到审阅（步骤 1）的情况：**
- 伙伴根据你的反馈更新了方案
- 基本方法需要重新思考

**不要强行突破阻塞** —— 停下并询问。

<constraints>
- 禁止跳过验证步骤
- 禁止在方案要求引用技能时忽略
- 禁止阻塞时猜测——必须停下询问
- 禁止未经用户同意在 main/master 分支上实现
- 禁止只更新 TodoWrite 而不回写 plan 文件——plan 文件是唯一真相来源
</constraints>

## 集成

**必需的工作流技能：**
- **claude-workflow:worktree** —— 确保隔离工作区（创建一个或核验已有的）
- **claude-workflow:plan** —— 编写本技能要执行的方案
- **claude-workflow:finish** —— 在所有任务后收尾开发
