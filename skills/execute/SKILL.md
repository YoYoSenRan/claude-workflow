# 执行方案

<HARD-GATE>
每完成一步都要运行对应验证，确认结果后再继续。

没有用户明确同意，不要在 main/master 分支上动手。
</HARD-GATE>

## 触发

- 已有批准的书面方案。
- 用户选择在当前会话逐步执行。
- 任务强依赖当前上下文，不适合拆给独立代理。

如果方案适合拆成独立任务，并且当前环境支持子代理，主控可以选择 subagent。execute 不负责重新决定整体路线。

## 先做

1. 确认是否需要 worktree；已经隔离则跳过。
2. 找到方案目录；用户未指定时，从 `docs/plans/` 中找有待办任务的 README.md。
3. 读取 README.md 和当前 task 文件。
4. 如果已有完成项，从第一个未完成任务继续。
5. 发现方案缺口、冲突或看不懂的步骤，先问用户。

## 执行

对每个任务按顺序做：

1. 标记 TodoWrite 为 in_progress。
2. 读取场景和步骤。
3. 按步骤改文件。
4. 运行该步骤的验证命令或人工核对。
5. 通过：把 task checkbox 改成 `[x]`，README.md 状态改成 ✅，TodoWrite 标记 completed。
6. 失败：在 task 文件记录失败信息，README.md 状态改成 ❌，停止并报告。

README.md 是进度源；TodoWrite 只辅助当前会话。

## 完成后

1. 对照所有场景和 diff 自检。
2. 按方案或风险调用 review。
3. 调用 verify，拿到完成声明证据。
4. 只有用户要提交、PR、合并、保留或丢弃时，才进入 finish。

## 停止

- 依赖缺失。
- 测试或验证失败。
- 方案说明不清。
- 实际代码和方案假设冲突。
- 连续修复仍失败。

## 输出

停止时输出：
- 当前任务
- 已改文件
- 验证命令和结果
- 阻塞点
- 需要用户决定什么

完成时输出：
- 完成的任务
- 验证证据
- 剩余风险或未做项

## 禁止

- 不许跳过验证。
- 不许只更新 TodoWrite 而不更新 README.md。
- 不许卡住后猜着做。
- 不许未经同意在 main/master 上动手。
