# 子代理笔记格式合同

子代理 P2 阶段交付物必须遵守的格式。主代理依赖此格式做 P3 合并。

## 三段结构

### ① 来源段

```
[X1] {作者/机构} — {标题} | URL | 类型: {标签} | 日期: YYYY-MM-DD | 权威: 1-10 | 状态: {✓/⚠/✗}
```

字段含义 —

| 字段 | 必填 | 说明 |
|---|---|---|
| `X` | ✓ | 任务字母 (A/B/C...), 配合编号唯一 |
| 作者/机构 | ✓ | 原始出处, 不是聚合站 |
| 标题 | ✓ | 实际页面标题 |
| URL | ✓ | 真访问过的完整链接, 不准缩短/编造 |
| `类型` | ✓ | 见下表 |
| `日期` | ✓ | 文章发布日期 (找不到精确日, 至少给年月) |
| `权威` | ✓ | 权威度 1-10, 见评分卡 |
| `状态` | ✓ | 访问性标签, 见下表 (规约在 fetch-methods.md) |

**状态取值**:

| 标签 | 含义 | 入注册表? | 权威度影响 |
|---|---|---|---|
| `✓ 畅通` | WebFetch 直拉成功 | ✓ | 无 |
| `⚠ 代理 (r.jina.ai)` | 经 Jina 代理拿到 | ✓ | 自动 -1 |
| `⚠ 代理 (defuddle.md)` | 经 Defuddle 代理拿到 | ✓ | 自动 -1 |
| `⚠ 快照 (YYYY-MM-DD)` | archive.org 历史快照 | ✓ | 自动 -1, 标快照日期 |
| `✗ 受限` | 全链失败 | ✗ | 不计入, 进缺口段 "访问失败" |

**类型取值**:

| 标签 | 含义 | 例子 |
|---|---|---|
| `官方` | 第一手官方 | 公司博客、SEC 文件、政府报告、GitHub 官方仓库 |
| `学术` | 同行评议 | 期刊论文、会议论文、博士论文 |
| `行业分析` | 专业分析 | 行业研报、分析师覆盖、专业媒体深度文 |
| `新闻` | 新闻报道 | 主流媒体、调查报道 |
| `社区` | 用户生成 | 论坛、评论、社交媒体、Q&A 站 |
| `其他` | 混合/未明 | 聚合站、未核实源 |

**权威度 1-10 评分卡**:

| 分 | 标准 |
|---|---|
| 9-10 | 一手官方 + 强信誉 (公司官方 + 政府/学术) |
| 7-8 | 知名媒体 / 行业头部分析师 / 高星 GitHub 维护中项目 |
| 5-6 | 一般行业博客、中型媒体、活跃个人专家 |
| 3-4 | 营销页、聚合站、内容农场 |
| 1-2 | 匿名贴、内容明显有偏 |

### ② 发现段

```
- {单句事实} [X1][X3]
- {单句事实} [X2]
```

规则 —
- 每条**单句**, 不要"另外/此外"扩展
- 每条**至少 1 个 `[Xn]`**, 多源支撑就多列
- 数字 + 日期 + 单位齐全
- 不确定标 `[未核实]` 或不写
- ≤15 条, 多了说明粒度太细
- 优先放可量化、可对比的事实, 而非感受性描述

### ③ 缺口段

至少**一条具体内容**, 覆盖以下任一类:

```
- 搜了但没找到: {具体什么问题没答案}
- 反向解释候选: {如果当前结论错, 可能错在哪}
- 时效风险: {哪些源已 >18 月}
- 访问失败: {标 ✗ 受限的 URL 列表 + 简述原因, 如"LinkedIn 反爬 / Crunchbase 登录墙"}
```

**缺口不是缺点 — 是诚实的边界声明**。允许写"无明显缺口"但必须紧接说明"已交叉验证 N 条主张, 未发现对立证据"等具体依据 — **不接受单字"无"**。

## 可选段: 深度精读

仅深读任务才有。2-3 篇全文蒸馏:

```
### [X1] {标题}
- **关键数据**: {3-5 条带数字的事实}
- **核心观点**: {作者主张}
- **反例/限制**: {哪些场景作者承认不适用}
```

## 主代理校验清单

收到子代理笔记后, 主代理过一遍 (单任务阈值, 与 SKILL.md P2 同步) —

- [ ] 来源段: 深度模式 4-8 条 / 常规模式 3-5 条
- [ ] 每条来源字段齐全 (URL / 类型 / 日期 / 权威 / **状态**)
- [ ] ⚠ 代理 / ⚠ 快照 的源已自动权威度 -1
- [ ] ✗ 受限的源没出现在来源段 (应进缺口段"访问失败")
- [ ] 发现段每条都有 `[Xn]` 源号
- [ ] 类型不全是 `社区` 或 `其他`
- [ ] 至少有一条 `官方` 或 `学术`
- [ ] 有缺口段, 且含至少一条具体内容 (非单字"无")

不达标 — 把笔记发回, 让子代理补 (或主代理自己补一轮搜索)。

注: 全局 (注册表) 阈值在 SKILL.md P3 质量阀。本表只管单任务。
