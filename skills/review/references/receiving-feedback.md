# 接收评审反馈

收到用户或外部评审反馈时使用。反馈是待验证的建议，不是自动执行命令。

## 执行

1. 读完整反馈。
2. 按条目列出要处理的问题。
3. 不清楚的条目先问清楚；不要先做一部分。
4. 对每条反馈检查代码、测试、需求或已有决策。
5. 分类：采纳 / 回推 / 需要用户决定 / 无法验证。
6. 逐条修复；每修一条跑对应验证。
7. 最后汇报处理结果和验证证据。

## 来源处理

| 来源 | 处理 |
|---|---|
| 用户 | 理解后执行；范围不清先问 |
| 外部评审员 | 先验证是否适合当前代码库 |
| 自动工具 | 核对是否真实问题，避免机械改动 |

## 回推条件

- 建议会破坏现有功能。
- 评审员缺少完整上下文。
- 建议违反 YAGNI。
- 技术上不适合当前栈或版本。
- 与用户已有架构决策冲突。

## 输出

```text
反馈处理：
- 采纳：...
- 回推：...
- 需要确认：...
- 无法验证：...

验证：
- ...
```

## 禁止

- 不许表演式认同。
- 不许验证前直接实施外部反馈。
- 不许部分理解就开始修。
- 不许批量修完才测试。
- 不许把外部评审当成用户决定。
