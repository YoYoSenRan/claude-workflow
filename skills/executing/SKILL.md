# 执行实现计划

加载 `plan` 产出的计划文件, 评审 → TodoWrite → 逐任务执行 → 阻塞点就停。

<HARD-GATE>
遇到阻塞点 (测试失败 / 依赖缺失 / 指令不清 / 实现跟计划不符) → **停**, 报告用户。

禁止: 跳步骤 / 自己改计划 / 猜测意图 / "差不多就行"。

计划是契约, 不是建议。
</HARD-GATE>

## 反模式: "我看一眼计划就开干"

最常掉的坑。直接跳到 Task 1 = 漏掉关键评审 = 带 bug 上路。

计划写错的概率 > 0。先评审再勾清单。

## 清单

按顺序完成:

1. **Load** — 读计划全文 (Read 工具, 一次性)
2. **关键评审** — 找阻塞点 (歧义 / 缺依赖 / 顺序倒挂)
3. **TodoWrite 建** — 每任务 = 一项
4. **逐任务执行**:
   - 标为 in_progress
   - 按步骤走, 验证 expected
   - 标为 completed
5. **遇阻塞点停** — 不猜不试, 报告用户
6. **完工** — 全部 done → 提示后续 (finishing)

## 流程图

```
Load 计划
   ↓
关键评审 ── 有阻塞点 ──→ 报告用户, 暂停
   ↓ 通过
TodoWrite 建项
   ↓
取下一任务
   ↓
标为 in_progress
   ↓
按步骤执行 + 验证
   ↓
通过? ── 否 ──→ 阻塞点? ── 是 → 停, 问
   ↓ 是                     └ 否 → 修, 重试 (最多 1 轮)
标为 completed
   ↓
还有任务? ── 是 ──→ 取下一
   ↓ 否
提示后续 (finishing)
```

## 关键评审

加载计划后, 动手前, 静默问自己:

- [ ] 任务顺序无依赖倒挂?
- [ ] 文件路径都存在? (Create 例外)
- [ ] 命令能跑? (依赖装了吗?)
- [ ] 步骤之间无遗漏?
- [ ] 我能理解每步骤干啥?

任一答"否" → 阻塞点, 报告用户, 不动手。

## 阻塞点类型 + 处理

| 阻塞点 | 处理 |
|---|---|
| 测试失败超 1 轮重试 | 停, 报告: "任务 N 步骤 M 测试持续失败, 输出: ..." |
| 文件 / 命令未找到 | 停, 报告: "任务 N 假设 X 存在, 实际不存在" |
| 步骤含糊不能落地 | 停, 报告: "任务 N 步骤 M 含糊, 候选解读: A / B" |
| 实现跟规格不符 | 停, 报告: "任务 N 实现 X, 但计划写的是 Y" |
| 修改超出计划范围 | 停, 报告: "任务 N 需改文件 Z, 但计划没列" |

**不要做的**:

- ✗ 跳过失败步骤继续下一任务
- ✗ 自己改计划
- ✗ 猜测用户意图
- ✗ 编 expected 输出

## TodoWrite 用法

加载计划后立刻建项:

```
TodoWrite([
  { content: "Task 1: <name>", status: "pending" },
  { content: "Task 2: <name>", status: "pending" },
  ...
])
```

每任务开始 → 标为 `in_progress`; 完成 → 标为 `completed`。

步骤级别**不**进 TodoWrite (粒度太细, 用计划自己的清单)。

## TDD 处理

计划怎么写, 你怎么执行。

- 计划第 1 步骤是"写失败测试" → 老实写, 跑, 看到红
- 计划没测试步骤 → 不要自己加 (违反计划即契约)

要问的: 计划该不该有测试? → 这是计划阶段的事, 不归你。

## 前后衔接 (软引用, 不强制)

- **前序**: 计划产出的 `docs/plans/*.md`
- **后续**: 全部任务完成 → 跑测试看新鲜证据 + 自己 `git commit` + 提 PR

本仓**不计划补** finishing skill (见 README ROADMAP), 收尾保持轻量, 由你自行做。

## 警示信号

| 内心戏 | 真相 |
|---|---|
| "步骤大概是这意思, 直接干" | 大概 ≠ 确定。停, 问 |
| "测试失败可能环境问题, 跳过" | 跳过 = 隐藏 bug |
| "这步骤没写代码, 我补一下" | 补 = 改计划。停, 问 |
| "计划漏了 X, 我顺手加" | 顺手 = 范围扩散。停, 问 |
| "我先把简单任务全做了" | 顺序破坏 = 依赖倒挂风险 |
| "阻塞点等会再处理" | 现在就停, 继续 = 累积错误 |
| "TodoWrite 太重, 不建了" | 建。进度可追踪 = 用户能审 |

## 核心原则

- **HARD-GATE 不可破** — 阻塞点必停
- **计划即契约** — 不擅自改
- **不猜不试** — 模糊就问
- **TodoWrite 必建** — 进度可追踪
- **逐项验证** — 每步骤 expected 必跑
