# 交付决策

<HARD-GATE>
验证没通过，不许给合并、PR 或清理选项。

没有用户明确确认，不许删除分支、移除工作树、丢弃提交或强制推送。
</HARD-GATE>

## 触发

- 用户明确要提交、合并、创建 PR、保留分支或丢弃成果
- execute 或 subagent 完成后，用户要处理分支/工作树
- 需要决定如何整合工作成果

普通完成汇报不进入 finish。

## 执行

1. 确认本轮已有新鲜验证证据；没有就先验证。
2. 验证失败时停止，不展示交付选项。
3. 检查当前仓库状态：

   ```bash
   git status --short
   git branch --show-current
   git rev-parse --show-toplevel
   git rev-parse --git-dir
   git rev-parse --git-common-dir
   ```

4. 如果当前是 worktree，确认它是不是本流程创建的。不是自己创建的，不清理。
5. 给用户列交付选项。
6. 只执行用户选择的选项。

## 选项

正常分支：

1. 本地合并回 `<base-branch>`
2. 推送并创建 PR
3. 保留当前分支/工作区
4. 丢弃这次工作

游离 HEAD 或外部管理工作区：

1. 推送为新分支并创建 PR
2. 保留当前工作区
3. 丢弃这次工作

## 执行动作

**合并：**
- 切到目标分支前确认没有未提交改动会丢失。
- 合并前拉取最新目标分支。
- 合并后重新运行验证。
- 合并成功且验证通过后，才考虑清理工作树和删除分支。

**推送并创建 PR：**
- 推送当前分支。
- 创建 PR，正文必须包含 summary 和 test plan。
- 不清理工作树，保留给 PR 反馈迭代。

**保留：**
- 报告当前分支、路径、验证命令和状态。
- 不清理任何东西。

**丢弃：**
- 先列出将永久删除的内容：分支、提交、工作树路径。
- 要求用户输入精确的 `discard`。
- 未收到精确确认前，不执行任何删除。

## 输出

- 当前分支和路径
- 验证证据
- 用户选择
- 执行结果
- 保留的风险或后续动作

## 禁止

- 验证失败时不许创建 PR、合并或清理。
- 不许开放式问"接下来做什么"；必须给结构化选项。
- PR 后不清理 worktree。
- 不许清理外部管理的 worktree。
- 不许在未确认时丢弃成果。
