# Search v2 (极简版)

派子代理搜资料 → 主代理整合成给同事看的调研笔记。**写工程笔记, 不写学术论文**。

## 流程 (2 步)

### 1. 派遣
拆 2-3 个子任务并发派子代理 (Agent tool, subagent_type=general-purpose), 按 [references/subagent.md](references/subagent.md) 模板。每任务 1 个专家角色, 5 分钟内完成。

### 2. 起草
子代理回笔记后, 主代理整合:
- 直接答用户原题, 第一句就给结论, 不要 "本调研发现"
- 段末统一 `(参考 N)`, 不要每句末挂 `[n]`
- 数据来源放末尾, **必须 `1. 2. 3.` 数字编号**, 1 行 1 源
- 写完自己读一遍 — 啰嗦 / 重复 改掉

## 开门一句话 — 必须给主推荐

用户问"调研 X 选哪个 / 对比 X / 综述 X" 时, **第一句必须明说"首选 Y" 或"选 Y"**, 不要四档分类摆烂。

✓ **好**: "Vue3 后台首选 vue-vben-admin v5 (32k 星 + 多 UI 内核可换), EP 党选 vue-pure-admin。"
✗ **坏**: "国内中后台默认 EP + pure-admin 或 vben v5; 看脸选 soybean; 想脱锁选 shadcn-vue。"

差别 — 好例子开口就拍板, 坏例子四档没主见。**用户调研就是要个推荐, 不是看流派综述**。

如果真没法拍板 (候选并列), 用 "X / Y 二选一, X 适合 A, Y 适合 B" 句式 — 仍是明确推荐, 不是 4+ 档摊开。

## 输出风格 (硬性)

| 要 | 不要 |
|---|---|
| 开门一句直答 + 主推荐 | TL;DR / 摘要 / 本调研发现 / 四档摆烂 |
| 人话段名: 重点 / 对比表 / 选哪个 / 盲点 / 数据来源 | 学术词: 决策矩阵 / 关键发现 / 引文注册表 / 附录 / 核心争议 / 局限 / 没拿到的 |
| 对比表用 markdown 或 ASCII 框都行 | (用户接受 ASCII, 不强制) |
| 自然语言信心度 ("3 个官方源, 数据可靠") | "整体信心度: 中-高" 评级 |
| "数据截至 2026 年 5 月" 一句 | 列"模式 / 视角 / 主意图" 元数据 |
| 段落自然引文 (参考 1, 3) | `→ [N]` 箭头 |

## 结构

按问题类型自己判断, 不强制模板。**只有 3 段必备**:
1. **开门一句话** (直答 + 主推荐)
2. **正文** (按主题或对比, 自己定)
3. **数据来源** (`1. 2. 3.` 编号, 1 行 1 源, 末尾一句可信度)

其他段 (重点 / 对比表 / 选哪个 / 盲点 / 怎么试) — **有内容才出**, 没内容整段省。

### 盲点段 (原"没拿到的", 旧名易误解)

调研中实际没拿到的数据 / 访问受限的源 / 时效风险, 让读者知道边界:

```
## 盲点
- {访问失败的源} (反爬 / 登录墙 / 全链失败)
- {数据空白} (Bundlephobia 挂, 没拿到 gzip 体积)
- {时效风险} (源 > 18 个月)
```

**≤ 5 条单句**, 多了说明子代理没尽力。0 个盲点时整段省。

### 数据来源段 (必备)

```
1. {作者} - {标题} - URL
2. {作者} - {标题} - URL
3. ...
```

**必须数字编号** — 引文 `(参考 1, 3)` 严格对应到 `1.` `3.`, 否则读者没法溯源。

末尾一句自然语言信心度: "数据可信度: 大半官方源, 选型够用; 商业模板价格需自行复核当时定价页"。

## 抓取失败

WebFetch 失败按 [references/fetch.md](references/fetch.md) 走代理级联, 不死磕重试。

## 不打断用户

启动后 (派遣开始) 一律自动决策跑完, 不问用户。失败按级联走, 实在不行末尾"盲点"段说明。

只有 1 种情况打断: **主题歧义** (用户提的名词指向多个) → 派遣前一次问完。

## 适合 / 不适合

| 适合 | 不适合 |
|---|---|
| 多源对比 (库对比 / 选型) | 单页问答 (用 WebFetch) |
| 综述 / 行业分析 / 政策研究 | 写文章 / 润色 |
| 调研报告需可追溯引用 | 单文件理解 (用 Read 工具) |

## 内部规则 (子代理用, 不露给用户)

- 子代理笔记里可标 `✓ 畅通` / `⚠ 代理` / `⚠ 快照` / `✗ 受限` 状态
- 主代理起草时**翻译成自然语言**: ⚠ 代理 → "二手摘录", ✗ 受限 → 进"盲点"段
- 报告里不出现 `✓` `⚠` `✗` 这些状态符号

## 自查 (起草完 30 秒)

写完报告自己再读一遍, grep 这些字面命中即改:
- `TL;DR` / `决策矩阵` / `关键发现` / `引文注册表` / `核心争议` / `附录` (学术词)
- `→ [` (旧式引文)
- `✓ 畅通` / `⚠ 代理` / `✗ 受限` (内部状态符号, 报告里改自然语言)
- `没拿到的` (旧段名, 改"盲点")

**ASCII 框 (`┌─┐` `├─┤`) 用户已接受, 不强制改**, 但宽表 (>6 列) 仍建议拆。

数据来源段必须 grep 出 `^1\.` `^2\.` 等数字开头, 否则缺编号。

不强制核验清单, 自己读着顺不顺即可。
